## コンポーネント詳細

### vLLM（ホスト直接実行）

**理由**: GPUドライバへの直接アクセスが必要なため、Docker内では動かさない。

| 項目 | Primary | Secondary |
|---|---|---|
| モデル | Llama 4 Scout 109B Q4_K_M | Qwen 3.5 32B Q5_K_M |
| ポート | 8080 | 8081 |
| VRAM | 55GB (0.58) | 22GB (0.23) |
| コンテキスト | 32768 tokens | 16384 tokens |
| 用途 | 複雑推論・コード・人格 | 高速・日本語会話 |

### LiteLLM Proxy（Docker）

OpenAI互換のAPIゲートウェイ。チーム全員とAntGravityが `gpt-4o` というエイリアスで
そのままローカルLLMを使える。

```
gpt-4o       → Llama 4 Scout :8080
gpt-4o-mini  → Qwen 3.5 32B  :8081
claude-sonnet → Anthropic（フォールバック）
```

## データフロー

1. クライアントが `POST /v1/chat/completions` を送信（`Authorization: Bearer mdl-llm-2026`）
2. Nginx (:80) → LiteLLM (:8000) にプロキシ
3. LiteLLM がモデルエイリアスを解決してvLLMに転送
4. vLLM がBlackwell GPU上で推論を実行
5. SSEストリームまたはJSONでレスポンスを返す

## 起動順序

```
1. vLLM Primary  (bash vllm/start_primary.sh)
2. vLLM Secondary (bash vllm/start_secondary.sh)
3. Docker services (docker compose up -d)
   → LiteLLM → Prometheus → Grafana → Nginx
```

vLLMが先に起動している必要がある（LiteLLMがヘルスチェックで確認するため）。
